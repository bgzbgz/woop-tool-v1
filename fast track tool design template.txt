## üé® VISUAL IDENTITY

### Color Palette
- **Primary Background**: `#FFFFFF` (White)
- **Primary Text**: `#000000` (Black)
- **Secondary/Borders**: `#E0E0E0` (Light Grey)
- **Context Boxes**: `#F8F8F8` (Off-white)
- **Accent Borders**: `#000000` (Black, 4px solid for emphasis)

**Design Philosophy**: Clean, minimal, professional. White canvas with black and grey accents only.

## üìù TYPOGRAPHY

### Font System
1. **Headings Font**: `Plaak` (Bold)
   - Font file: `Plaak3Trial-43-Bold.otf`
   - Usage: All headings, section titles, numbered sections
   - Properties: `font-weight: bold; letter-spacing: -0.01em; line-height: 1.2;`

2. **Body Font**: `Riforma` (Regular)
   - Font file: `RiformaLL-Regular.otf`
   - Usage: All body text, inputs, buttons, descriptions
   - Properties: `font-size: 16px; font-weight: normal;`

### Typography Scale
- **Large Headings (WHY/WHAT/HOW)**: `36px` (Plaak)
- **Page Title**: `32px` (Plaak)
- **Section Numbers** (e.g., [01]): `18px` (Plaak) with `padding: 10px 16px`
- **Subheadings**: `20px` (Plaak)
- **Body Text**: `16px` (Riforma)
- **Small Text**: `14px` (Riforma)
- **Button Text**: `16px` (Riforma)

---

## üèóÔ∏è LAYOUT PRINCIPLES

### 1. **Worksheet-Style Layout**
- Full visibility of related sections
- Left sidebar for context (WHY/WHAT/HOW instructions)
- Main content area for input/interaction
- Collaborative team meeting focus (not survey-style)

### 2. **Information Architecture**
- ‚úÖ **DO**: Show related information together
- ‚úÖ **DO**: Use accordions for expandable sections (if >3 items)
- ‚úÖ **DO**: Progress sequentially (lock future sections until current is complete)
- ‚ùå **DON'T**: Show only one input at a time (unless specifically a survey flow)
- ‚ùå **DON'T**: Create visual overwhelm with too many sections visible at once

### 3. **Spacing System**
- Section padding: `24px` (p-6)
- Element spacing: `16px` (space-y-4, gap-4)
- Tight spacing: `8px` (gap-2)
- Large spacing: `32px` (space-y-8)

## ‚ú® INTERACTION PATTERNS

### 1. **Progressive Disclosure**
- Lock sections until prerequisites are complete
- Show completion status clearly
- Use "Next" buttons to guide progression

### 2. **Instant Feedback**
- Show validation states immediately
- Use subtle animations (0.2s ease transitions)
- Highlight active/focused elements with black border

### 3. **Placeholders & Hints**
- Use varied, helpful placeholder text
- Provide 3-5 different examples per input type
- Make hints actionable and specific

### 4. **Animations**
```
Standard Transition: transition: all 0.2s ease;
Hover States: Subtle scale or background change
Modal Entry: Slide from center, fade in backdrop
Accordion: Smooth height transition
```
